GitHub设置PAT/SSH

发表于2021-09-12|SoftWare
|字数总计:1.1k|阅读时长:4分钟

从2021-08-13起Github的Git Operations Over HTTPS不再支持使用Web登录密码验证，不过可以使用Personal Access Token(PAT)代替密码，但有一个缺点：PAT有效期最长90天，如果 不想每隔一段时间更换Token也可以使用SSH的方式。

1 HTTPS

1.1 设置PAT

按照官方文档即可设置PAT，由于自动生成的PAT长度可达40位，而且关闭网页后不再显示，所以请首先复制PAT到本机。

设置好之后，我们仍可以使用https的方式clone or push代码，只不过提示输入密码的时候我们输入PAT即可，如果不嫌麻烦可以每次在命令行里输入这一长串字符串。

当然也可以不用每次都输入这令人头疼的PAT，解决办法就是Windows的凭据管理。

1.2 Windows凭据管理

打开我的电脑，在地址栏输入：控制面板\用户帐户\凭据管理器，然后回车即可打开凭据管理。

密码的作用是保护私钥，每次使用私钥前需要先输入正确的密码。

打开我的电脑，在地址栏输入：%USERPROFILE%/.ssh，即可看到公钥：id_ed25519.pub，私钥：id_ed25519。私钥保存在本机，公钥需要添加到GitHub，公钥与私钥都可以直接用记事本之类的文本工具打开查看。

2.2 GitHub添加SSH公钥

登录GitHub后，点击右上角的头像，然后依次选择Settings->SSH and GPG keys->New SSH key or Add SSH key。

Title可以输入一个名称用以区分不同的公钥，Key对应着公钥，将id_ed25519.pub中的内容复制到里面即可。

也可参照官方文档。
